About Chinook

Chinook is a sample database containing information that one might find in a music store, with relations such as tracks, artist, genres, and invoices. This web app tech stack includes MariaDB (MySQL server), fetchAPI for the front end, and a node.js server. This was created to explore the database with a more intuitive UI and as practice working with these technologies. Thanks for visiting!

The Track Table

The Track table represents the tracks of this music store. Each track possesses a unique Track ID, a name, the ID of the album it belongs to, an ID describing what media type it is (eg. MP3), an ID describing what genre it is, a composer, a length in milliseconds, a size in bytes, and a unit price. A track can be in several Playlists and a Playlist can have several tracks, so it is a M:N relationship. Tracks have a 1:N relationship with albums, an album can have N amounts of tracks but every track just has one album.
